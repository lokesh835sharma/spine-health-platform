About SpineAI

A portfolio-grade machine learning healthcare platform demonstrating end-to-end ML deployment — from data collection to real-time prediction.

🎯

Project Goal

SpineAI was developed to demonstrate the application of unsupervised machine learning in healthcare. The project covers the complete ML lifecycle: survey design, data collection, preprocessing, model training, evaluation, and production deployment through a full-stack web application.

The platform is designed as a portfolio showcase for Amazon ML Summer School, internship applications, and GitHub portfolios — demonstrating real-world skills in Python, Flask, scikit-learn, PCA, and K-Means clustering.

Algorithms Used

Primary

PCA (Principal Component Analysis)

Reduces 12 preprocessed features to 2 principal components for K-Means clustering and visualization. PC1 captures pain/symptom variance; PC2 captures lifestyle variance.

Primary

K-Means Clustering (k=4)

Unsupervised clustering using K-Means++ initialization. Optimal k determined by Elbow Method. Clusters manually labeled based on feature profiling.

Explored

DBSCAN

Density-based clustering explored as an alternative. Found less suitable due to dataset density characteristics and sensitivity to epsilon parameter.

Explored

Hierarchical Clustering

Agglomerative clustering with Ward linkage. Dendrogram analysis confirmed the 4-cluster structure, providing additional validation for K-Means results.

Explored

Gaussian Mixture Models

Probabilistic clustering model explored for soft assignments. Similar cluster structure identified, confirming robustness of the 4-cluster framework.

Metrics

Evaluation Metrics

Silhouette Score (separation + compactness), Davies-Bouldin Index (overlap), and Calinski-Harabasz Index (separation vs compactness ratio) used for evaluation.
